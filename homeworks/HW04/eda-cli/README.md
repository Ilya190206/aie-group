В рамках обновления EDA-CLI инструмента были добавлены новые возможности для более гибкой генерации отчётов. Основные команды остались прежними: overview для быстрого просмотра структуры датасета и report для создания полного аналитического отчёта в формате Markdown с графиками.

К команде report добавлены три новых опциональных параметра, которые позволяют кастомизировать процесс анализа:

    --top-k-categories — управляет количеством наиболее частых значений, выводимых для каждой категориальной колонки. Это помогает детальнее или, наоборот, лаконичнее представить распределение категорий в данных.

    --title — позволяет задать произвольный заголовок для генерируемого отчёта, что удобно для документирования различных итераций анализа.

    --min-missing-share — задаёт пороговое значение доли пропусков (от 0.0 до 1.0). Все колонки, в которых доля пропущенных значений превышает этот порог, будут выделены в специальный раздел «Проблемные колонки» для привлечения внимания аналитика.

Пример вызова с новыми параметрами:



python -m src.eda_cli.cli report data/dataset.csv \
    --title "Еженедельный анализ пользовательской активности" \
    --top-k-categories 10 \
    --min-missing-share 0.05

Эта команда сгенерирует отчёт под заданным заголовком, покажет топ-10 значений для категориальных полей и выделит все колонки, где более 5% данных пропущено.
##  HTTP API (FastAPI)

Сервис предоставляет следующие эндпоинты:

### Основные эндпоинты:

1. **GET /health** - Проверка здоровья сервиса
2. **POST /quality** - Оценка качества из JSON
3. **POST /quality-from-csv** - Оценка качества из CSV файла

### Новые эндпоинты (HW04):

4. **POST /quality-flags-from-csv** - Полный набор флагов качества
   - Включает все новые эвристики из HW03
   - Возвращает детальную информацию о проблемных колонках
   - Пример ответа:
     ```json
     {
       "flags": {
         "has_constant_columns": true,
         "has_high_cardinality_categoricals": false,
         "has_suspicious_id_duplicates": true,
         "has_many_zero_values": false,
         "quality_score": 0.85
       },
       "metadata": {...}
     }
     ```

5. **POST /dataset-summary** - Полная сводка датасета в JSON
